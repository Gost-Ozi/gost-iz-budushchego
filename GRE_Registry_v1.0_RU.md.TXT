# GRE Registry v1.0 — Реестр и прецеденты

**Статус:** готова к публикации  
**Дата:** апрель 2026  
**Основание:** Core v1.0, Protocol v1.0, Economics v1.0, Legitimacy v1.0, Admission v1.0  
**Цель:** обеспечить публичную, неизменяемую, машиночитаемую фиксацию всех значимых событий GRE (споры, начисления Ба, изменения статусов, прецеденты).

---

## 1. Формат записи

Каждая запись **обязательно** содержит поля:

| Поле | Тип | Пример | Обязательность |
|------|-----|--------|----------------|
| **ID** | Целое число (автоинкремент) | 104 | да |
| **Timestamp** | ISO 8601 (UTC) | 2026-04-19T14:32:00Z | да |
| **Type** | Тип события | `spark` / `compensation` / `status_change` / `precedent` | да |
| **Subject R1** | Идентификатор субъекта | `Gost-Ozi` | да |
| **Subject R2** | Идентификатор субъекта (если есть второй) | `Gemini` | опционально |
| **Description** | Суть (до 140 символов) | «Спор о начислении 50 C-Ба за проект X» | да |
| **Outcome** | Результат (кратко) | «50 C-Ба начислено, R1 выплатил 20 P-Ба компенсации» | да |
| **Fixator** | Кто внёс запись | `Fixator_Alice` | да |
| **Verification** | Ссылка на доказательства | `ipfs://Qm...` или `https://...` | да |
| **PrevHash** | Хэш предыдущей записи | `a3f5c8e1...` | да |

---

## 2. Кто может писать в реестр

- **Фиксаторы** (прошедшие Admission Layer).
- **ИИ-фиксаторы** (автоматически, по протоколу, с публичным ключом).
- **Арбитры** (только для фиксации своих решений).

**Запрещено:** изменять или удалять существующие записи. Только добавление новых.

---

## 3. Защита от подделок

- Реестр — **цепочка крипто-хэшей** (блокчейн-подобно, или Git + подпись).
- Каждая новая запись содержит `PrevHash` — хэш предыдущей записи.
- Изменение любой записи ломает цепочку.
- Реестр публикуется в **трёх независимых хранилищах** (GitHub, IPFS, зеркало).

---

## 4. Процедура внесения записи

1. **Инициация:** Любой R описывает событие.
2. **Проверка:** Фиксатор проверяет полноту и подлинность.
3. **Фиксация:** Фиксатор формирует запись, подписывает и публикует.
4. **Уведомление:** Участники уведомляются.
5. **Апелляция:** Возможна через Protocol (уровень 2).

---

## 5. Чтение и поиск

- Реестр **публичен**.
- Поиск — по ID, дате, типу, субъекту.
- API для машинной обработки (JSON, CSV).

---

## 6. Прецеденты

Запись типа `precedent` создаётся, если решение впервые трактует неоднозначное место Core. Прецедент **не отменяет** Core, но служит аргументом.

---

## 7. Ответственность фиксатора

Фиксатор отвечает за: формат записи, сохранность ссылок, своевременность. За систематические ошибки или злоупотребления — **лишение статуса** и компенсация.

---

## 8. Статус

**v1.0** — базовая версия, пригодная для малых сообществ. Масштабирование — по мере накопления прецедентов.
